The Problem

AI agents were proliferating faster than enterprises could manage them.

Marketing spun up a Microsoft Copilot agent. Supply Chain deployed an Amazon Bedrock agent. IT built a custom troubleshooting bot. Each team moved fast — and independently. Within months, a large enterprise could have hundreds of AI agents running across different platforms, with no central visibility into what they were doing, what data they were touching, or whether any of them were failing silently.

This is Agent Sprawl — and it's the operational chaos that happens when AI adoption outpaces AI governance.

"The CIO's nightmare: no idea how much those agents cost, no audit trail if a financial agent authorised the wrong transaction, no way to shut down a rogue agent before it caused damage."

The Product

One control plane. Every agent. Everywhere.

Boomi Agent Control Tower (ACT) is the centralised governance and control plane within Boomi Agentstudio. If Orchestrate is the conductor pulling systems and agents into workflows, the Agent Control Tower is the security and air traffic control team watching over every AI agent running across the enterprise.

See how Agent Control Tower works ↗

Three pillars defined the product:

01

Universal Governance — No Walled Gardens

ACT is vendor-agnostic. IT doesn't just monitor agents built natively in Boomi — they can register, catalog, and control agents across the entire tech stack: Amazon Bedrock, Microsoft Copilot, Salesforce, Snowflake Cortex, and custom homegrown agents via GraphQL APIs. One single pane of glass for every piece of AI in the enterprise.

02

Proactive Monitoring & Anomaly Detection

Rather than waiting for an agent to fail or go rogue, ACT continuously tracks execution against a 30-day baseline behaviour window. If an agent spikes in latency, token usage, or error rates — or takes an unusual reasoning path — the Control Tower flags it immediately for IT to intervene before damage is done.

03

Enterprise Guardrails & Policy Enforcement

Before an AI agent can execute a workflow, it must comply with central guardrails — role-based access controls, data permissions, and audit logging of every agent interaction, tool usage, and prompt instruction. For supported providers like Amazon Bedrock, administrators can trigger a kill-switch to instantly disable a rogue agent.

My Role

What I owned

I worked as Product Owner alongside 1 PM, 2 EMs and 15+ developers, owning three specific areas of the ACT product:

Audit & Activity Log

Every agent interaction, tool call, reasoning path, and system event needed to be captured and surfaced in a way that made compliance audits actually usable — not just a raw data dump. I owned the design and delivery of the audit log experience.

Providers Management

The multi-provider capability — connecting Bedrock, Copilot, Salesforce, Snowflake, and custom agents into a single governance layer. I owned this end to end, including the GraphQL API integration for custom providers.

Graphical & List Interface

Enterprise IT admins needed two modes: a high-level visual map of all agents running across the organisation, and a detailed list view for operational management. I owned both — balancing visual clarity with granular control.
